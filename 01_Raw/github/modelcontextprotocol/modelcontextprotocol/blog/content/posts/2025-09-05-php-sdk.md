The official [PHP SDK](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/PHP SDK) for the Model Context Protocol is now generally available.

Built in collaboration with the [PHP Foundation](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/PHP Foundation) and [Symfony](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/Symfony), the PHP SDK handles protocol details, so developers don’t have to worry about low-level mechanics and can focus on building their applications.

The initial release enables PHP developers to build MCP [servers](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/servers), exposing [tools](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/tools), [prompts](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/prompts), and [resources](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/resources) to AI applications. Support for PHP applications to act as MCP [clients](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/clients) will follow.

The PHP SDK now joins 9 other [officially supported language SDKs](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/officially supported language SDKs) in the MCP ecosystem, making it easier for developers everywhere to adopt MCP in their preferred language.

## Get involved

The PHP SDK is now open to the community to install, test, and contribute:

- SDK repo: [modelcontextprotocol/php-sdk](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/modelcontextprotocol/php-sdk)
- Composer package: [mcp/sdk](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/mcp/sdk)

We welcome your feedback and contribution, including issues, documentation improvements, and pull requests. Framework-specific integrations and real-world examples are also particularly valuable.

## Thanks to the MCP community

This release consolidates earlier community work into a single, trusted implementation. The SDK is maintained by the [Symfony](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/Symfony) team, with [Kyrian Obikwelu](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/Kyrian Obikwelu) joining as a maintainer based on his previous [PHP-MCP](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/PHP-MCP) work. The [PHP Foundation](https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/blog/content/posts/PHP Foundation) helped to coordinate the initiative with support from the members of MCP steering group.

Thank you to all involved in bringing PHP to the MCP ecosystem.
