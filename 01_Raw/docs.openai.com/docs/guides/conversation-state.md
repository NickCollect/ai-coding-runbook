# Conversation state

<!-- source: https://platform.openai.com/docs/guides/conversation-state -->

OpenAI provides multiple ways to manage conversation state across multiple turns.

## 1. Manually manage conversation state

Pass previous messages as parameters in your text generation request. Include the model's previous response output as input, then append to your next request.

```python
response = client.responses.create(
    model="gpt-4o-mini",
    input=[
        {"role": "user", "content": "knock knock."},
        {"role": "assistant", "content": "Who's there?"},
        {"role": "user", "content": "Orange."},
    ],
)
```

## 2. Conversations API

The [Conversations API](https://platform.openai.com/api/docs/api-reference/conversations/create) works with the Responses API to persist conversation state as a long-running object with its own durable identifier. After creating a conversation, you can keep using it across sessions, devices, or jobs.

```python
conversation = openai.conversations.create()

response = openai.responses.create(
    model="gpt-4.1",
    input=[{"role": "user", "content": "What are the 5 Ds of dodgeball?"}],
    conversation="conv_689667905b048191b4740501625afd940c7533ace33a2dab"
)
```

Conversation objects and items are persisted with no 30-day TTL (unlike response objects which expire after 30 days).

## 3. `previous_response_id` parameter

Chain responses by passing the ID of a previous response:

```python
response = client.responses.create(
    model="gpt-4o-mini",
    input="tell me a joke",
)

second_response = client.responses.create(
    model="gpt-4o-mini",
    previous_response_id=response.id,
    input=[{"role": "user", "content": "explain why this is funny."}],
)
```

Even with `previous_response_id`, all previous input tokens for responses in the chain are billed as input tokens.

## Context window management

- Output tokens: tokens generated by the model (max depends on model)
- Context window: total tokens for input + output (+ reasoning tokens for reasoning models)
- `gpt-4o-2024-08-06`: 16,384 max output, 128K context window
- Use the tokenizer tool to estimate token counts

### Compaction

For long conversations, use compaction to manage context:
- Server-side: `context_management` parameter with `compact_threshold`
- Manual: `/responses/compact` standalone endpoint

## Data retention

- Response objects saved for **30 days** by default (viewable in dashboard logs)
- Set `store: false` to disable retention
- Conversation objects: no 30-day TTL
- OpenAI does NOT use API data to train models without explicit consent
