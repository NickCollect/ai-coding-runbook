## 一句话定义

Claude Code 的可复用专项能力，包含 SKILL.md + 资源文件，按上下文自动加载

## 关键属性

- Skill = 文件系统 artifact：`<name>/SKILL.md` (YAML frontmatter + Markdown body) + 可选 `scripts/` `references/` `assets/` 子目录 [[skills]] [[skill-development--skill]]
- 遵循 [agentskills.io](https://agentskills.io) 开放标准；Claude Code 在此之上加了 invocation control / subagent execution / dynamic context injection 三类扩展 [[skills]]
- 三层 progressive disclosure：metadata（始终在 context，~100 词）→ SKILL.md body（触发后，建议 <500 行 / 5k 词）→ bundled resources（按需加载，scripts 可执行而无需进 context） [[SKILL--skill-creator]] [[skill-development--skill]]
- 存储路径四种：enterprise managed > personal `~/.claude/skills/<name>/` > project `.claude/skills/<name>/` > plugin `<plugin>/skills/<name>/`；同名时 enterprise > personal > project，plugin skill 自带 `plugin-name:skill-name` 命名空间不冲突 [[skills]]
- 同名时 **skill 优先于 slash command**；老的 `.claude/commands/foo.md` 与新的 `.claude/skills/foo/SKILL.md` 都创建 `/foo`，但 skill 拿到调用 [[skills]]
- 触发方式两种：用户 `/skill-name` 显式调用 / Claude 根据 description 自动加载（model-invoked，autonomous） [[skills]] [[skills--agent-sdk]]
- SKILL.md frontmatter 关键字段：`name` / `description`（必填，决定 trigger 命中率）、`when_to_use` / `argument-hint` / `arguments` / `allowed-tools` / `model` / `effort` / `context: fork` / `agent` / `hooks` / `paths` / `shell` / `disable-model-invocation` / `user-invocable` [[skills]]
- `description` + `when_to_use` 合计上限 1,536 字符；description char budget = context window 1%，floor 8000 char，可用 `SLASH_COMMAND_TOOL_CHAR_BUDGET` 覆盖 [[skills]]
- 描述写作要"略带 pushy"——Claude 倾向 under-trigger skill；用 imperative 第三人称 ("This skill should be used when...")，解释 WHY 比 ALL-CAPS MUST 更有效 [[SKILL--skill-creator]] [[skill-development--skill]]
- 字符串替换变量：`$ARGUMENTS` / `$ARGUMENTS[N]` / `$N` / `$name` / `${CLAUDE_SESSION_ID}` / `${CLAUDE_EFFORT}` / `${CLAUDE_SKILL_DIR}`（plugin-portable 脚本路径） [[skills]]
- Dynamic context injection：`` !`<command>` `` 在 Claude 看到内容前执行，输出替换 placeholder（preprocessing，非 Claude 执行）；可用 settings 的 `disableSkillShellExecution: true` 全局禁用，bundled/managed skills 不受影响 [[skills]]
- Subagent 集成两种 pattern：`context: fork` 让 skill 在 forked subagent context 跑（skill body 变 subagent prompt）；或 subagent 的 `skills` 字段预加载 skill 给自定义 subagent [[skills]] [[skills--agent-sdk]]
- Lifecycle：被调用时整份 SKILL.md 渲染入会话作为一条 message 并停留整个 session，**不会**后续轮次重读；auto-compaction 时每个 skill 最近一次调用的前 5,000 token 重附，所有 re-attached skills 共享 25,000 token 预算 [[skills]]
- Live change detection：`~/.claude/skills/`、project `.claude/skills/`、`--add-dir` 加入的 `.claude/skills/` 目录均被 watch，添加/编辑/删除 session 内即时生效；新建顶层 skills dir 需重启 [[skills]]
- 限制 Claude 访问：`/permissions` deny Skill tool 全禁；`Skill(name)` / `Skill(name *)` 单独 deny；`disable-model-invocation: true` 从 Claude context 完全移除；`user-invocable: false` 仅隐藏菜单不阻止程序化访问 [[skills]]
- SDK 行为：discovery 经 `settingSources` (TS) / `setting_sources` (Python)，默认加载 `user`+`project`；启动时只读 metadata，body 在 invoke 时加载；必须 `"Skill"` 在 `allowedTools` 才启用；**SDK 忽略 SKILL.md 的 `allowed-tools` 字段**，CLI 才 honor [[skills--agent-sdk]]
- Bundled resources 三类用途：`scripts/` 可执行（deterministic / token-efficient，可 execute 不进 context）；`references/` Claude 按需 load（>10k 词加 grep pattern 进 SKILL.md）；`assets/` 用于输出（templates / icons / fonts），不进 context [[skill-development--skill]]
- Anthropic 通过 `anthropics/skills` repo 作为参考实现 + plugin marketplace（`anthropic-agent-skills`），打包三个 plugin：`document-skills`（xlsx/docx/pptx/pdf，source-available 非开源）、`example-skills`（algorithmic-art / brand-guidelines / canvas-design / mcp-builder / skill-creator 等 12 个）、`claude-api` [[skills--repo-readme]] [[skills--marketplace]]
- 内建 bundled skills 每 session 都有：`/simplify` / `/batch` / `/debug` / `/loop` / `/claude-api` 等 [[skills]]
- 与 plugin command 集成：command 文本提到 skill name 即提示 Claude 调用，要求 skill 真实存在于 plugin 的 `skills/` 目录 [[plugin-features-reference]]

## 相关

- [[Plugin]] — Skill 是 plugin 内部最常打包的能力单元；plugin 的 `skills/` 目录被自动发现
- [[Slash-command]] — Skill 与 slash command 同名时 skill 优先；二者共享 `/name` 触发语法
- [[Subagent]] — Skill 可通过 `context: fork` 在 subagent context 中运行；subagent 反向也可用 `skills` 字段预装 skill
- [[Plugin-marketplace]] — `anthropics/skills` 既是参考实现也是官方 marketplace，分发 document-skills / example-skills / claude-api 三个 bundle
- [[Agent-SDK]] — SDK 通过 `settingSources` 发现 skill；行为与 CLI 有差异（`allowed-tools` 字段被忽略）
- [[Hooks]] — SKILL.md 可声明 skill-scoped hooks，控制 skill 生命周期事件
- [[Settings]] — `disableSkillShellExecution` 等管控开关位于 settings.json
