## 一句话定义

Claude Code 可分发安装的扩展包，可包含 skills/commands/hooks/MCP servers/subagents/output styles

## 关键属性

- 自包含目录，可打包 skills / agents / hooks / MCP servers / LSP servers / monitors / output styles / themes，通过 marketplace 跨 project 共享 [[plugins]] [[plugins-reference]]
- 必须有 `.claude-plugin/plugin.json` manifest（plugin 根目录唯一允许放在 `.claude-plugin/` 里的文件），其余组件必须放 plugin **根目录**而非 `.claude-plugin/` 下 [[plugins]] [[plugins-reference]]
- Manifest 必填字段只有 `name`（kebab-case，正则 `/^[a-z][a-z0-9]*(-[a-z0-9]+)*$/`）；可选 metadata 包含 `version`/`description`/`author`/`homepage`/`repository`/`license`/`keywords`/`$schema` [[manifest-reference--plugin-structure]] [[plugins-reference]]
- `version` 走 semver，omit 时用 git commit SHA，等于"每次 commit = 新版本"模式；显式写则必须 bump 才能让用户更新 [[plugins-reference]] [[plugin-marketplaces]]
- 组件 path 必须 relative + 以 `./` 开头，不允许 `../` 或绝对路径，统一 forward slash [[manifest-reference--plugin-structure]]
- Skills 在 plugin 内**强制 namespaced**：`/plugin-name:skill-name`，避免 collision；commands 子目录会再加一层 namespace [[plugins]] [[plugin-features-reference]]
- Plugin 内 `agents/` 不允许配置 `hooks` / `mcpServers` / `permissionMode`（安全限制），但可设 `name`/`description`/`model`/`effort`/`maxTurns`/`tools`/`disallowedTools`/`skills`/`memory`/`background`/`isolation: "worktree"` [[plugins-reference]]
- 两个核心 env var：`${CLAUDE_PLUGIN_ROOT}`（plugin 安装路径，每次 update 会变）和 `${CLAUDE_PLUGIN_DATA}`（`~/.claude/plugins/data/{id}/`，跨 update 持久化，用于 node_modules / venv / cache）[[plugins-reference]] [[plugin-features-reference]]
- 安装命令 `claude plugin install <plugin>[@marketplace] [-s scope]`；scope 包括 `user`（默认）/ `project` / `local`（gitignored）/ `managed`（read-only） [[plugins-reference]] [[discover-plugins]]
- 本地测试用 `claude --plugin-dir ./my-plugin`（可重复传），同名时本地覆盖已安装版本；`/reload-plugins` 不重启 pick up 改动 [[plugins]]
- Marketplace plugin 安装时被 copy 到 `~/.claude/plugins/cache`（每个 version 独立目录），旧版本 update/uninstall 7 天后标 orphaned；path traversal `../shared-utils` 在 cache 里失效，workaround 是用 plugin 根目录内的 symlink [[plugins-reference]]
- Plugin 间依赖通过 `dependencies` 数组声明（v2.1.110+），支持 semver range `~2.1.0` / `^2.0` / `>=1.4` / `=2.1.0`；cross-marketplace 依赖默认 blocked，要在 root marketplace 的 `allowCrossMarketplaceDependenciesOn` 显式允许 [[plugin-dependencies]]
- Tag release 命令 `claude plugin tag --push`，tag 格式 `{plugin-name}--v{version}` 必须与 plugin.json 一致；cache dir 名带 12-char commit SHA suffix 防止 tag force-move [[plugin-dependencies]]
- Agent SDK 加载 plugin 仅支持 `type: "local"` + filesystem path，远程/marketplace 必须先下载；init message 的 `message.plugins` 和 `message.slash_commands` 用来验证加载 [[plugins--agent-sdk]]
- Background monitors（`monitors/monitors.json`，v2.1.105+）是 persistent 进程，stdout 每行 → Claude notification；plugin 启用时 auto-start，run 时 unsandboxed 在 hooks trust level [[plugins]] [[plugins-reference]]
