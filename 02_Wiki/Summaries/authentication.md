How to log into Claude Code and configure auth for individuals, teams, orgs, and CI. First launch of `claude` opens a browser; if browser cannot reach localhost (WSL2, SSH, containers), copy-paste a code returned by the browser into the prompt.

**Account types**:
- Claude Pro / Max subscription (claude.ai)
- Claude for Teams / Enterprise (admin invite). Enterprise adds SSO, domain capture, RBAC, compliance API, managed policy settings.
- Claude Console (API-billed). Admin invites users via Console Settings → Members or SSO. Roles: **Claude Code** (only Claude Code API keys) vs **Developer** (any key).
- Cloud providers: Amazon Bedrock, Google Vertex AI, Microsoft Foundry — set env vars, no browser login.

`/logout` and `/login` switch identity. `/status` shows the active method.

**Credential storage**:
- macOS: encrypted Keychain
- Linux/Windows: `~/.claude/.credentials.json` (or `$CLAUDE_CONFIG_DIR`); Linux file mode `0600`; Windows inherits user profile ACL
- Supported types: Claude.ai, Claude API, Azure, Bedrock, Vertex auth

**`apiKeyHelper`** setting runs a shell script returning an API key — useful for vault-fetched short-lived tokens. Default refresh: every 5 min or on HTTP 401. Override via `CLAUDE_CODE_API_KEY_HELPER_TTL_MS`. Helper >10s shows a "slow helper" warning. **`apiKeyHelper`, `ANTHROPIC_API_KEY`, `ANTHROPIC_AUTH_TOKEN` apply only to terminal CLI sessions** — Desktop and remote sessions are OAuth-only.

**Auth precedence (highest first)**:
1. Cloud provider creds (`CLAUDE_CODE_USE_BEDROCK` / `_USE_VERTEX` / `_USE_FOUNDRY`)
2. `ANTHROPIC_AUTH_TOKEN` → sent as `Authorization: Bearer` (LLM gateway use)
3. `ANTHROPIC_API_KEY` → sent as `X-Api-Key`. Interactive mode prompts once; non-interactive `-p` always uses it
4. `apiKeyHelper` script output
5. `CLAUDE_CODE_OAUTH_TOKEN` env var (long-lived OAuth, generated by `claude setup-token`)
6. `/login` subscription OAuth (default for Pro/Max/Team/Enterprise)

**Common gotcha**: with an active subscription AND `ANTHROPIC_API_KEY` set, the API key wins once approved; if the key's org is disabled/expired, auth fails. Fix: `unset ANTHROPIC_API_KEY`. Verify via `/status`.

**Long-lived tokens for CI**: `claude setup-token` walks through OAuth and prints a 1-year token; export as `CLAUDE_CODE_OAUTH_TOKEN`. Requires Pro/Max/Team/Enterprise. Inference-only — cannot establish Remote Control sessions. **Bare mode (`--bare`) does NOT read `CLAUDE_CODE_OAUTH_TOKEN`** — use `ANTHROPIC_API_KEY` or `apiKeyHelper` for those scripts.

Claude Code on the Web always uses subscription credentials; sandbox env vars cannot override.
