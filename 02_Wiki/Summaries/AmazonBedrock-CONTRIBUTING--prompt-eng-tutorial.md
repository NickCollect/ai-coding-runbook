Standard AWS-style contributing guide for the Bedrock Edition of the Prompt Engineering Interactive Tutorial.

**Reporting bugs / feature requests.** Use GitHub issue tracker. Check existing issues first. Useful details: reproducible test case or steps, version of code, modifications made, environment/deployment specifics.

**Pull requests.** Work against latest source on `main`. Check existing/recently merged PRs. Open an issue to discuss significant work first.

**PR steps.** Fork the repo. Modify only the files relevant to the change (avoid broad reformatting that obscures it). Ensure local tests pass. Commit to fork with clear messages. Send a PR and answer the PR template questions. Follow up on automated CI failures and reviewer comments.

The file continues with code-of-conduct, security-issue reporting, and licensing sections — boilerplate AWS sample-repo contribution guide. Identical in spirit to the same file in the `courses/` repo's Bedrock subdirectory.
