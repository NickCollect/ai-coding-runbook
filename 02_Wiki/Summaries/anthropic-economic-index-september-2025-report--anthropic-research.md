第三版 Anthropic 经济指数报告（2025 年 9 月，基于 2025 年 8 月数据）首次纳入 150+ 国家和全美各州的地理分析，以及首份企业 API 使用数据，核心发现是 AI 采用高度集中于少数地区和任务，且与收入水平强烈正相关。

**使用时序变化**（V1→V3，约八个月）：
- 编程任务仍占主导（36%），但教育类任务从 9% 升至 12%，科学类从 6% 升至 7%
- "directive"（完全委托）模式从 27% 跃升至 39%，首次出现自动化使用超过增强使用
- 代码创建任务占比增加 4.5pp（反映模型可靠性提升），调试任务下降 2.8pp

**Anthropic AI 使用指数（AUI）**：以各地区 Claude 使用份额除以其劳动年龄人口份额计算。全球 AUI 与 GDP 正相关，GDP 每增 1% 对应 Claude 使用量增 0.7%。以色列（AUI=7）、新加坡（4.57）、澳大利亚（4.10）领先；印度（0.27）、印度尼西亚（0.36）、尼日利亚（0.2）偏低。美国内部：DC（3.82）、犹他州（3.78）领先，加州（2.13）第三。

**地理差异与任务多样化**：低采用率国家使用高度集中于编程（印度超过 50%），高采用率国家使用更多样化（教育、科学、商业运营均有体现）。高采用率国家在控制任务结构后更倾向于增强模式（augmentation），低采用率国家更倾向于自动化（automation），原因尚不明确。

**企业 API 使用（首次披露）**：
- 77% 为自动化模式（vs Claude.ai 约 50%），高度集中于编程和办公/行政任务
- API 用户对成本不敏感：高成本任务反而使用量更大；控制任务特征后，成本每增 1% 仅对应使用量减少 0.29%
- 复杂任务对上下文长度有更高要求（输入长度弹性 0.38），表明"上下文可及性"可能是企业部署瓶颈，而非模型能力本身

**政策含义**：AI 生产力收益若集中于已富裕地区和自动化友好行业，可能加剧全球经济不平等；早期职业工作者面临更大位移风险，而拥有隐性组织知识的资深工作者可能受益于 AI 补充效应。
