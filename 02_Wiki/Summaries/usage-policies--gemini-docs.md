# Abuse Monitoring and Usage Policies

Source is in Simplified Chinese (crawler localization).

## Overview

Google monitors Gemini API and AI Studio for abuse via automated + manual processes. Users agree to these policies + [Gemini API Additional Terms](https://ai.google.dev/gemini-api/terms) + [Generative AI Prohibited Use Policy](https://policies.google.com/terms/generative-ai/use-policy).

## Monitoring Methods

- **Automated detection**: Automated systems scan API usage for violations (hate speech, harassment, sexual content, dangerous content)
- **Manual review**: Projects with persistent suspicious activity may be flagged for manual review by authorized Google personnel

## Data Retention for Abuse Monitoring

Google retains the following for **55 days** to detect and prevent policy violations:
- **Prompts**: Text prompts submitted to the API
- **Context**: Additional context provided in prompts
- **Output**: Responses generated by Gemini API

## Human Review Process

When prompts or outputs are flagged by safety filters:
- Authorized Google employees may evaluate flagged content
- Only authorized employees can access data for manual review through internal governance platforms
- Data used exclusively for policy enforcement and abuse prevention
- **NOT used to train/fine-tune other AI/ML models** (except models specifically for policy enforcement)

## Developer Responsibility

By using Gemini API, developers agree to:
1. Monitor their own applications for policy compliance
2. Maintain appropriate safeguards against misuse
3. Promptly address policy violations reported by Google
