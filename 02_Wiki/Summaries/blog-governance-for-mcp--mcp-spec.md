David Soria Parra (Lead Maintainer) introduces MCP's formal governance model after rapid post-launch growth strained the original ad-hoc process.

**Specification Enhancement Proposals (SEPs)** become the primary mechanism for proposing protocol changes. Inspired by Python PEPs and Rust RFCs, SEPs are submitted as GitHub issues (note: this was later changed in November 2025 to PR-based — see `blog-2025-11-28-sep-process-update`). Workflow: submit proposal → maintainers review and tag for sponsorship → contributor works with sponsor + community through draft/review/implementation stages.

**Three leadership roles** are established:

- **Maintainers** — manage specific components (SDKs, docs, individual repos)
- **Core Maintainers** — guide overall project direction and evolution of the spec
- **Lead Maintainers** — final decision-makers ensuring long-term project health

All maintainers form the **MCP steering group**. Core and Lead Maintainers meet bi-weekly to review SEPs; meeting notes are public.

**Get-involved paths** for new contributors (good first issues), SDK developers, documentation writers, future maintainers (path starts with consistent quality contributions). Discord and GitHub are the main coordination channels.

Closes with thanks to early adopters, client/server developers, and existing maintainers — explicitly framing future governance as preserving open inclusive community spirit while bringing process clarity.
