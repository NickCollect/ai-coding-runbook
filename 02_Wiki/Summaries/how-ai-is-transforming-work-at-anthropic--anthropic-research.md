Anthropic 对内部 132 名工程师和研究人员的调查与访谈显示：AI 正在大幅提升生产效率并拓宽技能边界，但同时引发技能萎缩、社交减少和职业不确定性等深层隐忧。

**调查方法**：2025 年 8 月，通过问卷（132 人）、深度访谈（53 人）和 20 万条 Claude Code 内部记录（对比 2025 年 2 月与 8 月数据）三角验证，是 Anthropic 首次系统性自我研究。

**生产效率**：员工自报将 Claude 用于 59% 的工作（去年仅 28%），平均获得 50% 的生产效率提升（去年约 20%）；14% 的"重度用户"生产效率提升超过 100%。27% 的 Claude 辅助工作是此前根本不会完成的新任务（如交互式数据仪表盘、代码重构"小改善"等）。

**Claude Code 使用趋势**：任务复杂度从 3.2 升至 3.8（5 分制），Claude 连续工具调用次数从 9.8 次增至 21.2 次（+116%），人工干预轮次减少 33%；实现新 feature 的使用占比从 14% 增至 37%。

**技能变革**：工程师普遍"全栈化"（backend 工程师开始构建 UI，研究员自建可视化工具），但同时担忧深层技能萎缩——"监督悖论"：有效监督 Claude 需要恰恰是可能因 AI 而退化的编程技能。

**社交动态变化**：Claude 成为问问题的首选（80-90% 的问题先问 Claude），高级工程师收到的初级员工提问减少，传统师徒制受到冲击。

**职业不确定性**：许多工程师描述自己已转型为"AI agent 管理者"，但对长期前景普遍存在不确定性；有人表示"每天上班感觉像在帮自己失业"，也有人保持乐观。

**研究局限**：样本存在选择偏差（积极用户更可能参与调查）；自报数据存在社会期望偏差；研究时间点为 Claude Sonnet 4 和 Opus 4 时代，当前情况已进一步演变。
