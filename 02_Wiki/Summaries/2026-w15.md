Weekly digest of Claude Code releases v2.1.92 → v2.1.101.

**Headline features**:
- **Ultraplan** (research preview): plan mode in the cloud from your terminal. `/ultraplan <task>` drafts plan in a Claude Code on the web session while CLI stays free; review/comment in browser, choose remote execution or send back to CLI. v2.1.101 auto-creates default cloud env on first use.
- **Monitor tool** (v2.1.98): built-in tool that spawns a background watcher and streams events into the conversation as new transcript messages. Use for tailing training runs, watching CI, dev-server crashes — replaces Bash sleep loops.
- **`/loop` self-pacing**: omit interval and Claude schedules next tick based on task, or substitutes Monitor tool to skip polling.
- **`/autofix-pr`** (CLI): infers open PR for current branch and enables auto-fix on Claude Code on the web. Push branch → run command → walk away; Claude watches CI + reviews and pushes fixes until green.
- **`/team-onboarding`** (v2.1.101): generates teammate ramp-up guide from your local Claude Code usage in a project you know well.

**Other wins**:
- Focus view: `Ctrl+O` in flicker-free mode collapses to last prompt + one-line tool summary with diffstats + Claude's final response.
- Bedrock and Vertex AI guided setup wizards on login screen (pick "3rd-party platform").
- `/agents` tabbed layout: Running tab shows live subagents with `● N running` count.
- Default effort level now `high` for API-key, Bedrock, Vertex, Foundry, Team, Enterprise users.
- `/cost` per-model + cache-hit breakdown for subscription users.
- `/release-notes` interactive version picker.
- Status line `refreshInterval` setting + `workspace.git_worktree` JSON input.
- `CLAUDE_CODE_PERFORCE_MODE`: Edit/Write fail on read-only files with `p4 edit` hint.
- OS CA cert store trusted by default (enterprise TLS proxies work without setup); `CLAUDE_CODE_CERT_STORE=bundled` to opt out.
- `CLAUDE_CODE_USE_MANTLE=1` for Mantle Bedrock endpoint.
- Hardened Bash permissions: backslash flags, env-var prefixes, `/dev/tcp` redirects, compound commands now prompt correctly.
- `UserPromptSubmit` hooks can set session title via `hookSpecificOutput.sessionTitle`.
