Consolidated coverage of the per-package `CHANGELOG.md` files in the TS SDK monorepo. All currently sit at v2 alpha (`2.0.0-alpha.1` and `2.0.0-alpha.2`); the changelogs are auto-generated by `@changesets/cli` from the `.changeset/*.md` notes (see `changesets--mcp-ts-sdk` summary for full content).

**`@modelcontextprotocol/core` 2.0.0-alpha.1** (key minor changes): refactor extracting task orchestration from Protocol into TaskManager (breaking — `taskStore`, `taskMessageQueue`, `defaultTaskPollInterval`, `maxTaskQueueSize` moved from `ProtocolOptions` to `capabilities.tasks` on `ClientOptions`/`ServerOptions`); fix unknown-tool error code to `-32602` and unknown-resource to `-32002` (`ProtocolErrorCode.ResourceNotFound` added); Standard Schema support for tool/prompt schemas (Zod v4 / Valibot / ArkType all work via `RegisteredTool.inputSchema`/`outputSchema`/`RegisteredPrompt.argsSchema`).

**`@modelcontextprotocol/client` 2.0.0-alpha.1**: WebSocketClientTransport removed (not a spec-defined transport — use stdio or Streamable HTTP); add `discoverOAuthServerInfo()` and unified discovery state caching; OAuth Resource Server discovery improvements.

**`@modelcontextprotocol/server` 2.0.0-alpha.1** (major change): unknown-tool/resource error code fix (per-spec error codes — see core changelog).

**`@modelcontextprotocol/express` / `fastify` / `hono` / `node`**: 2.0.0-alpha versions; primarily release-coordination patch bumps tied to `@modelcontextprotocol/server` upgrades. Minor changes also added Fastify adapter, Express OAuth RS glue.

**`@modelcontextprotocol/test-integration` 2.0.0-alpha.1**: removal of deprecated `.tool`/`.prompt`/`.resource` method signatures.

All changelogs follow the standard changesets-emitted format with PR numbers, commit hashes, contributor handles, and impact-level annotations.
