GitHub issue template (`.yml` form schema) for reporting documentation problems in the `anthropics/claude-code` repo.

**Title prefix**: `[DOCS]`. **Labels**: `documentation`.

**Form fields**:
- `doc_type` (dropdown, required) — Missing / Unclear / Incorrect / Typo / Missing examples / Broken links / Other.
- `location` (input, optional) — URL where issue encountered.
- `section` (input, required) — specific section/topic.
- `current` (textarea, optional) — quote current docs.
- `issue` (textarea, required) — what's wrong/missing.
- `suggested` (textarea, required) — suggested improvement (with sample text if possible).
- `impact` (dropdown, required) — High (prevents feature use) / Medium (hard to understand) / Low (minor confusion).
- `additional` (textarea, optional) — screenshots, related links, examples from other projects.

Standard GitHub issue form spec — no Claude-Code-specific behavior.
